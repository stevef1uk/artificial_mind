# Go Language RPi Chatbot

This is the Go-based implementation of the Raspberry Pi Chatbot. It manages audio recording, speech-to-text (Whisper), AI chat interaction (via HDN Server), and text-to-speech (MeloTTS).

## Prerequisites

### 1. Port Forwarding (Cluster to Network)
To allow the Raspberry Pi to talk to the HDN Chat Server running in your Kubernetes cluster, you'll need to expose the service to your local network. 

Run this command on the machine with `kubectl` access (e.g., your cluster control plane, `192.168.1.63`):
```bash
kubectl port-forward svc/hdn-server-rpi58 8081:8080 -n agi --address 0.0.0.0
```
This binds the internal API port `8080` to the external port `8081`, making it accessible at `http://192.168.1.63:8081`.

### 2. Environment Variables (`.env`)
Create a `.env` file in the `go-chatbot` directory (or its parent directory) on the Raspberry Pi. This tells the chatbot where to find its services.

Example `.env`:
```ini
# The address of the Kubernetes Chat API (your port-forwarded IP)
HDN_ADDR=http://192.168.1.63:8081

# Local or network addresses for STT and TTS services
LLM8850_WHISPER_HOST=http://localhost:8801
LLM8850_MELOTTS_HOST=http://localhost:8802
```
*Note: The script also relies on `SOUND_CARD_INDEX` which is automatically detected by `run.sh`, but you can explicitly set it here if needed.*

## Building and Deployment

1. **Build the executable for ARM64 (Raspberry Pi)**:
   Run this on your local development machine:
   ```bash
   GOOS=linux GOARCH=arm64 go build -o chatbot .
   ```

2. **Copy to the Raspberry Pi**:
   ```bash
   scp chatbot stevef@192.168.1.60:/home/stevef/
   scp run.sh stevef@192.168.1.60:/home/stevef/
   ```

## Running the Chatbot

On the Raspberry Pi, you must use `sudo` to run the chatbot wrapper script. This is required because the TTS audio files generated by the cluster are often written with `root` ownership in `/tmp`, and because the audio device locks require elevated privileges to manage effectively.

```bash
# SSH into the Raspberry Pi
ssh stevef@192.168.1.60

# Run the chatbot
sudo ./run.sh
```

### What `run.sh` does automatically:
- Kills any stuck audio processes (`sox`, `play`, `aplay`).
- Detects the correct ALSA Sound Card index for the WM8960 HAT.
- Sets the ALSA hardware volume/gain controls (Speakers: ON, Mic Gain: 70% to prevent clipping).
- Reads your `.env` configuration.
- Starts the `chatbot` binary.
